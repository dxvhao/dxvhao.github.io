      # 介绍

      多路云是一款可以安装在通用硬件上的高性能虚拟化管理系统，实现与物理机同样用户体验。 多路云安装在主机存储盘中，而windows是跑在多路云的终端中（即虚拟机），所以只要多路云的网络配置好，可以远程管理维护终端的一切，包括安装终端操作系统或是配置终端的网络等等。

      # 它是如何工作的？

      实际上多路云的启动过程，其过程就像启动一个操作系统一样，主板开机自动执行BIOS（基本输入输出系统）引导程序启动后加载多路云后台管理软件。 只有多路云后台管理都跑起来后，终端才可以正常的启动或关闭。 如果您之前有使用过虚拟机的话，了解主机与终端的启动先后就会非常容易，不了解也没关系。 只要明白需要启动终端，就要先把主机启动。

      可以理解为多路云计算机相比与普通计算机在操作系统之上是多了一层管理软件，管理层中可以创建或删除终端，把终端可以理解为就是一个windows。

      # 集群管理软件

      集群管理软件通常安装在管理员电脑、教师机上，也可运行于独立的服务器上。集群管理软件负责实现多台多路云主机及其虚拟机和镜像的统一批量管理和维护。

      # 主机、终端与集群的关系？

      终端运行在多路云管理软件上，而多路云管理软件运行在计算机上，用户可以通过多路云管理软件来管理上面运行的终端。 而集群通过网络连接所有的多路云主机，实现对加入集群的多路云主机获得时实状态并且也可以发送统一管理指令来达到批量管理。 集群软件的诞生初衷就是为了支持多路云计算机统一部署、管理。

      # 什么是镜像

      镜像是一种文件存储形式，可以把许多文件的磁盘做成一个镜像文件，方便导出给其它多路云主机使用。如果你了解过Ghost镜像就会很容易明白镜像的概念，基本上它们差不多，但多路云的镜像文件更安全且不像Ghost镜像文件那样容易损坏并且导入的速度更快，这一般取决于你的设备速度。

      多路云的镜像可以理解为就是一个虚拟硬盘，这个硬盘专供终端使用，将这个硬盘导出来就是一个镜像文件，导入到其它多路云机器中就是一块同样的硬盘，并且拥有同样的数据。

      # 什么是子镜像

      子镜像就是快照，应用于存储服务与虚拟机中的一种写时复制（Copy On Write）技术。 拥有快照功能的存储系统具有很高的灵活性。

      子镜像不占用空间，所有的数据都从父镜像中读取，在对子镜像写操作时，该写操作对应的新数据将存储在子镜像中，之后对该数据的读取将从子镜像读取新的数据。

      从上图中2-2中，终端在写入数据时，会直接写入到子镜像中，而在做系统更改等操作父镜像的操作时会直接在子镜像中写入复制新的数据到子镜像。而在读取数据时（图2-3）对于系统更改部分与其它数据从子镜像的新数据读取，剩下之外的数据依然从父镜像中读取。

      快照技术使得在父镜像中存储的数据不被更改的特征能够在大量的子镜像中共享，极大的节省了存储空间。 另一方面父镜像中的数据在对子镜像写操作时不会被更改，能够方便的实现还原、回滚等操作。

      # 还原

      多路云的还原功能在虚拟机运行时会生成一个临时快照，任何对系统的更 改都只会写入快照而不会影响原始镜像。在虚拟机关闭后，该临时快照会被删除，下次虚拟机启动的时候 再生成一个干净的临时快照。这种机制保证了每次虚拟机启动的时候都是与原始镜像一致的系统，有效保 障了系统安全，降低了维护工作量。
